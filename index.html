Vue 计算属性和侦听器

发表于 2020-02-08

计算属性和侦听器

计算属性

模板内的表达式(插值表达式等)非常方便使用，我们可以在里面放入变量和一些js逻辑,会让模板过于复杂且难以维护。
例如：

<div id = "app">{{firstName+lastName}}</div>

这段代码中,模板不再是简单的声明式逻辑。
对于任何复杂逻辑，都应当使用计算属性。
用计算属性的话，在dom层这样写

<div id = "app">{{fullName}}</div>

在js层这样写

var vm = Vue({
    el:"#app",
    data:{
        firstName:"李",
        lastName:"二",
    },
    computed:{
        fullName:function(){
            return this.firstName + this.lastName;
       }
    }
})

这样就已经成功的使用了计算属性来完成逻辑的运算

计算属性的getter和setter

计算属性默认只有getter,但是我们可以给他加个setter。
这样当你赋值时候，值就会传给setter方法。
示例可看官方文档

计算属性与方法的区别

计算属性是有内部缓存的，如果依赖的值没有改变，则会立即返回上次的计算结果，不会执行函数。如果依赖的值发生了改变，则会重新执行函数并更新缓存。
而方法在每次渲染时候，都会重新执行一次函数。

当需要计算的某个属性性能开销大时候，偏向于使用计算属性，避免不必要的性能开销。

侦听器

虽然计算属性在大部分情况下更合适，但有时候也需要一个自定义侦听器，
来响应数据的变化，当需要在数据的变化时候执行一些操作，就可以使用侦听器。
格式：
{ [key: string]: string | Function | Object | Array }
key:是需要监听的表达式,
string:对应回调函数，值也可以是方法名，或者包含选项的对象。

Vue实例会在实例化时调用$watch,遍历watch对象的每一个属性。

示例：

Var vm = Vue({
   // .......
    watch:{
        firstName:function(){
            this.fullName = this.fristName + this.lastName;
            }
        }
})

Vue Study02

发表于 2020-02-03 更新于 2020-02-04

Vue 第二天

MVP和MVVM

MVP

Model-View-Presenter
Model:数据层 例如通过ajax获取到的数据
View:视图层 简单理解为DOM层
Presenter: 逻辑处理层 业务逻辑
传统使用Jquery的代码一般就是MVP
缺点：对DOM层的操作过多
面向DOM编程

MVVM

Model 数据层
View 视图层
VM Vue内置
用了MVVM 代码主要是对M层和V层做改变
减少了对DOM层的操作
当 M层和V层有了改变
会通过VM层来更新数据
面向数据编程

列表渲染 v-for

我们可以用v-for 指令基于一个数组来渲染一个列表。
v-for 指令需要使用 item in items 形式的特殊语法,
items 是源数据数组 而item则是被迭代的数组元素的别名。
对应官方文档链接：
https://vuejs.bootcss.com/guide/list.html

事件处理 v-on

可以用v-on指令监听DOM事件，并在触发时运行一些JavaScript代码。
格式
v-on:click=”函数名或执行逻辑”
需要注意的是,函数需要在Vue实例里写的
对应官方文档链接:
https://vuejs.bootcss.com/guide/events.html

表单输入绑定 v-model

你可以用v-model 指令在表单元素上创建双向数据绑定。
他会根据控件类型自动选取正确的方法来更新元素
他负责监听用户的输入事件以更新数据，并对一些极端场景进行一些特殊处理

v-model 会忽略所有表单元素的 value、checked、selected attribute >的初始值而总是将 Vue 实例的数据作为数据来源。你应该通过 JavaScript 在组件的 data 选项中声明初始值。

对应官方文档链接：
https://vuejs.bootcss.com/guide/forms.html
